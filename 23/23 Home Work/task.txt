Задача: Расширить функциональность галереи.

Добавить интерактива используя возможности работы с событиями.
То есть вам необходимо взять результат того, что вы получили в прошлом
домашнем задании и расширить функциональность вашего приложения новыми пунктами.
Полное разъяснение ДЗ в видео. Здесь даны пункты в короткой форме для справки.

Пример приложения: Галерея

1.+ По ссылке выше доступен рабочий пример того что у вас должно получиться.
2.+ Для построения галлереи использовать подход с шаблонными строками.
3.+ Селектбокс с выбором количества и выбором типа галереи убрать.
То есть строить все изображения одним способом.
4.+ Добавлять новый элемент в галерею после нажатия на соответствующую кнопку.
5.+ Отображать количество доступных элементов.
6.+ При добавлении полностью всех элементов, сделать кнопку «Добавить» серого цвета.
7.+ Если пользователь все таки нажал на кнопку, выводить модальное окно
с сообщением что дальнейшее добавление невозможно.
8.*+ Сделать модальное окно не с помощью alert а с помощью
специального компонента bootstrap 3
9.*+ Реализовать механизм удаления картинки. Для этого использовать
патерн «делегация». То есть обработчик события должен быть не на каждой кнопке
«удалить», а на элементе верхнего уровня.
10.+ Расширить галерею сортировкой (селекбоксом). Содержит 4 пункта:
«По имени А- Я», «По имени Я — А», «Сначала новые», «Сначала старые».
11.+ При изменении выбора в селектбоксе галерея перестраивается автоматически.
Нажатие на кнопку не требуется. Строится то количество изображений,
которое было на экране. То есть если вы добавили 6 картинок то и после
изменения сортировки их должно быть 6. При первой загрузке по умолчанию
применяется фильтр. «По имени А- Я».
12.+ Значение фильтра сохраняется в localStorage. Таким образом после
перезагрузки страницы должен применяться последний выбранный фильтр.
